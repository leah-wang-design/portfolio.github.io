SalesChamp Invoice Feature

A fast-paced design process for SalesChamp invoice feature

Background

Saleschamp is a new start-up that aims to build a CRM tool with simple solutions for invoicing and quotes. It supports freelancers, small businesses, and online stores running on Shopify.

The team has collected some basic user research, gone through multiple idea brainstorming sessions, and has developed a sets of user stories. At the beginning of Invoice feature design, we decided to run a design sprint to quickly build and get some early feedback for the invoice feature.

Problem

How Might we help users create invoices and track status easily?

Our Challenge

The team wants to create a intuitive and clean app that allow users to manage and track invoices easily. Coordinating between technique limits and design needs became a big challenge in the process.

Role

UX Research, UX Design, UI Design, Interaction Design, Prototype

Team

One designer and Three software engineers

Tools

Figma, Hand Sketch

Quickly access invoice creation from homepage

Easily add products from your Shopify store

Set recurring invoices, schedule the invoices and track status

Sprint Process

  • Day 1- Map problems

    • Before the sprint starts, we want to make sure the long-term goal(6-8 months) for the product is clear: helps Shopify users create and manage invoices and quotes easily. For this sprint, the focus is on designing the invoice feature and quickly get some feedback.

  • Questions

    • The team has some initial research for invoice apps from user interviews and Shopify app store reviews. With the insights from the research, we listed a bunch of questions we want to answer about the invoice.

    Initial user research insights(click to enlarge)


  • What should we do to make invoices creating smoother and faster?
  • How should we help users track invoices and order status?
  • What are users looking for when they choose an invoice app?
  • What stage do the users need to see and edit the template?

  • Map and User journey

    • I mapped out the most important journey for different users of the invoice feature.

    Map for different roles(click to enlarge)


    We concluded one type of persona as our target audience for this feature: Meticulous Accountant Luke. Based on the interviews we conducted with the target users, I mapped out the user journey of Luke, to better communicate with stakeholders and have a clear picture of what we are trying to solve.

    Persona and User Journey(click to enlarge)
  • HMW Statement

    • By interviewing the experts, we wanted to put more focus on help users with tracking the status of the orders.

    How Might we help users create invoices and track status easily?

  • Day 2- Sketch solutions
  • Lightning Demos

    • I reviewed several different existing apps of primary and secondary competitors and made quick sketches of some of the screens/ideas that we can get inspired from.

    Design Inspirations(click to enlarge)
  • Quick Sketches:

    • With the some basic ideas and inspirations in mind, I sketched 8 variations of my most critical screen- the landing page for Invoice.

    Crazy 8's sketching(click to enlarge)
  • Solution Sketch

    • I picked a final screen that makes the most sense- it allows users to create new invoices directly on the landing page, and makes status easy to see and take actions on. Based on the screen, I also came up with a solution board- a three-panel storyboard to demonstrate the user flow for users to create invoices, track status, preview, and send the invoice.

    Selected Solution(click to enlarge)
  • Day 3- Story board

    • After walking through the sketches from Day 2 with the team, I made some modifications and created a user flow in which a user can create new invoices, set up recurring new invoices, and see the status on the home page.

    Storyboard(click to enlarge)
  • Day 4- High-fidelity and prototype

    • With the storyboard I had from Day3, I designed a series of high-fidelity UI by adopting Shopify's design system- https://polaris.shopify.com/. I chose green as the theme color, as green evokes growth and trustworthiness. And I picked blue as the complementary color since it's soothing, calm, and associable with tech and innovation.

    Step1: Create New Invoice from homepage and start with adding new customer
    Step2: Add line items from Shopify store by selecting from the suggestion list
    Step3: Set up recurring schedule, schedule and send, you are all set!
  • Day 5- Testing and learn

    • Finally the last day! I conducted unmoderated user testings by using Maze and received 5 responses. We also interviewed 2 friends who worked at small businesses and had experience with invoices.

    We were able to know the team needs to study more of the invoice status and payment methods we can provide with the users, whether the engineer have technical limits to provide the status, etc. Besides, I also put together some feedback for the interface design:



    Feedback for the interfaces (click to enlarge)
  • Next Step

    • After sharing the feedback with the team, the team is happy with the progress and some immediate feedback we are able to get from the sprint. We decided to revisit the status and payment methods after the engineering team making more progress on the payments and invoice tracking techniques.
